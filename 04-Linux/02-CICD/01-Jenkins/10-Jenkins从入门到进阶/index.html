本文最后更新于：2018年12月5日

Jenkins进阶入门

1. 安装

docker-compose.yml文件

# 作者：岑忠满
# 日期：2018年12月3日
# 功能：jenkins-BlueOcean
# 说明：建议用户名使用root(必须为Docker用户组)，挂载的.m2路径也需要相同用户名
version: '3'
services:
  jenkins:
    image: jenkinsci/blueocean
    user: root
    ports:
      - 8080:8080
    volumes:
      - ./data:/var/jenkins_home
      - /var/run/docker.sock:/var/run/docker.sock

然后，访问hostName:8080就能看到界面了

2. 部署第一个Github应用

2.1 创建并将Jenkinsfile放在工程跟路径下

pipeline {
    agent {
        docker {
            image 'maven:3-alpine'
            args '-v maven-repository:/root/.m2'
        }
    }
    stages {
        stage('Build') {
            steps {
                sh 'mvn -B -DskipTests clean package'
            }
        }
        stage('Test') {
            steps {
                sh 'mvn test'
            }
        }
        stage('Deliver') {
            steps {
                sh 'java -jar target/*.jar'
            }
        }
    }
}

2.2 在Jenkins的BlueOcean中给添加流水线

这时候，你的应用会在Github每次更新的时候自动构建啦。然而，肯定不只是这样的，这就叫进阶怕被打。。。

3. 构建特定分支

我们肯定不是要每个分支都构建的呀，只有特定版本才需要构建。

stage('Deliver for development') {
    when {
        branch 'development' // 这里就写特定的分支啦
    }
    steps {
        sh 'Do Something'
    }
}
stage('Deliver for development') {
    when {
        branch 'development' // 这里就写特定的分支啦
    }
    steps {
        sh 'Do Something'
    }
}

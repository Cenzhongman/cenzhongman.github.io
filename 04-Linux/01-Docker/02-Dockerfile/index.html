Dockerfile的使用

使用自己的Dockerfile

Dockerfile是一个脚本文件，每一条指令构建一层。

# 基础镜像
FROM nginx
# 使用RUN来执行一条命令，注意一条指令作为一层，可以使用 / 换行，使用 # 注释 
RUN echo '<h1>Hello, Docker!</h1>' > /usr/share/nginx/html/index.html

在Dockerfile所在文件夹执行

docker build -t nginx:v3 .
  • -t Name and optionally a tag in the ‘name:tag’ format

理解上下文

如果我需要的文件（Dockerfile和其他需要使用的文件）在```/home/ubuntu/```路径下，那么这个命令就该变成```docker build -t nginx:v3```文件来忽略文件或在文件夹。

当然，你可以指定Dockerfile是哪一个，如在/home/ubuntu/上一级目录中```docker build -t nginx:v3 -f ../imDockerfile /home/ubuntu/ ```

容器中的复制和容器外的复制

```dockerfile
# 这里的点表示上下文，不存在../这样的格式
COPY ./123.data/ /opt/

RUN cp /opt/123.data /opt/data/

使用来自URL的Dockerfile

docker build https://github.com/twang2218/gitlab-ce-zh.git#:8.14

用给定的 tar 压缩包构建

docker build http://server/context.tar.gz

命令剖析

FROM

  • 基础镜像
FROM nginx

RUN

  • 使用RUN来执行一条命令，注意一条指令作为一层，可以使用 / 换行，使用 # 注释 
RUN echo '<h1>Hello, Docker!</h1>' > /usr/share/nginx/html/index.html

COPY

  • 这里的点表示上下文，不存在../这样的格式，点可以不写，可以使用通配符 * ？ 等
  • 后面是是目标目录，可以是容器内的绝对路径，或者使用WORKDIR来指定的工作目录
  • 目标目录不存在会自动创建
  • 源文件的元数据将会保留
COPY ./123.data/ /opt/

ADD

  • 加强版COPY，源路径可以是URL，但是存在文件权限需要修改，压缩包需要解压等问题，不如直接使用wget
  • 在ADD会自动解压到指定目录
ADD ubuntu-xenial-core-cloudimg-amd64-root.tar.gz /

CMD(用于主进程)，推荐使用json

  • 指定不到命令启动时候的默认命令
CMD echo $HOME
# bash是使用-c参数执行的
CMD [ "sh", "-c", "echo $HOME" ]
  • 上述的命令是以sh为主进程的，sh执行结束，命令就结束了
CMD service nginx start
# ==
CMD ["sh", "-c", "service nginx start"]# 这个程序在sh执行为守护进程，进程开启，sh命令执行完毕，接着就结束了主进程sh，这是错误的
# 正确做法是直接执行可执行文件作为主程序执行，，并且要求前台
CMD ["nginx", "-g", "daemon off;"]

ENTRYPOINT

  • 场景一：让镜像变成像命令一样使用
  • 场景二：应用运行前的准备工作

ENV(全部)

  • 支持RUN的环境变量$HADOOP_HOME直接使用
  • 同时作为容器内的应用作为环境变量
  • 含有空格的值用双引号括起来 
ENV <key> <value>

ARG(不包含容器内)

  • 这里定义的参数可以使用
docker build --build-arg <参数名>=<值> # 覆盖参数

VOLUME(挂载匿名卷)

  • ? 这个路径放到了文件系统的哪里
  • ？这个命名卷是什么鬼
VALUME /data
# 使用命名卷mydada替代匿名卷
docker run -d -v mydata:/data xxxx

EXPOSE声明端口

EXPOSE 指令是声明运行时容器提供服务端口，这只是一个声明，在运行时并不会因为这个声明应用就会开启这个端口的服务。在 Dockerfile 中写入这样的声明有两个好处，一个是帮助镜像使用者理解这个镜像服务的守护端口，以方便配置映射；另一个用处则是在运行时使用随机端口映射时，也就是 docker run -P时，会自动随机映射 EXPOSE 的端口。

WORKDIR

  • RUN的运行环境的工作目录（当前目录）
  • 使用的是最新的WORKDIR

USER

  • 修改以后的用户

ONBUILD + 其他指令

  • 当前镜像执行时候并不会执行，在构建下一级镜像时这里的zh
FROM debian:jessie-backports

RUN apt-get update \
    && DEBIAN_FRONTEND=noninteractive apt-get install -y -t jessie-backports --no-install-recommends  openjdk-8-jre-headless ca-certificates-java \

本文最后更新于：2019年5月4日

Stack

  • 使用的数据结构：数组
  • 使用建议：不建议使用
    • 推荐使用Deque<Integer> stack = new ArrayDeque<Integer>();

Stack 类是在jdk1.1中就有的容器类方法，表示后进先出（LIFO）的对象堆栈。Stack的实现有两种方式，数组和链表，源码中使用数组实现

  • 源码分析：
// Stack是Vector的继承类
public
class Stack<E> extends Vector<E> {
    // 默认的构造器啥也没有
    public Stack() {
    }

    // 加到栈顶，位于数组的末尾
    public E push(E item) {
        addElement(item);
        return item;
    }

    // 从栈顶取出
    public synchronized E pop() {
        E       obj;
        int     len = size();
        obj = peek();
        removeElementAt(len - 1);
        return obj;
    }

    // 获得栈顶，但不取出
    public synchronized E peek() {
        int     len = size();
        if (len == 0)
            throw new EmptyStackException();
        return elementAt(len - 1);
    }

    // 获得元素到栈顶的距离
    public synchronized int search(Object o) {
        int i = lastIndexOf(o);
        if (i >= 0) {
            return size() - i;
        }
        return -1;
    }

    // 其他方法继承自Vector，如isEmpty()、add()等
}

而，官方建议使用ArrayDeque而不是Stack

Deque 接口及其实现提供了 LIFO 堆栈操作的更完整和更一致的 set，应该优先使用此 set，而非Stack类。例如：
Deque stack = new ArrayDeque();
